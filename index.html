素因数分解の話

素因数分解してくれるサービス
http://factordb.com/

自分のSSH鍵がリストされてないか確認してみる

# 公開鍵の modulus
ssh-keygen -m PKCS8 -e -f authorized_keys  > kagi
openssl rsa -pubin -inform PEM -text -noout < kagi > modulus

# 秘密鍵の modulus とか prime1, prime2の確認
openssl rsa -text -noout -in ~/.ssh/id_rsa

こんな感じのスクリプトで10進数にすればOK

RAID5のリビルド中にもう一個壊れるってやつ

リビルド中にもう一個壊れてRAID5が終わった話

/dev/md0:
        Version : 1.2
  Creation Time : Fri Jan  6 12:24:44 2012
     Raid Level : raid5
     Array Size : 5860535808 (5589.04 GiB 6001.19 GB)
  Used Dev Size : 1953511936 (1863.01 GiB 2000.40 GB)
   Raid Devices : 4
  Total Devices : 4
    Persistence : Superblock is persistent

    Update Time : Sat Oct 17 16:59:57 2015
          State : clean, FAILED
 Active Devices : 2
Working Devices : 3
 Failed Devices : 1
  Spare Devices : 1

         Layout : left-symmetric
     Chunk Size : 512K

           Name : hoge:0
           UUID : 55b9d6ea:4627c873:a0bcab05:a4f0c25d
         Events : 6755

    Number   Major   Minor   RaidDevice State
       5       8       49        0      active sync   /dev/sdd1
       2       0        0        2      removed
       2       8       33        2      active sync   /dev/sdc1
       6       0        0        6      removed

       4       8        1        -      faulty   /dev/sda1
       6       8       17        -      spare   /dev/sdb1

HDD一台が壊れたので /dev/sdb1を追加中にアレイの構成Diskがもう一個壊れた
無理やり起動してみる

mdadm -S /dev/md0
mdadm -A /dev/md0 -u 55b9d6ea:4627c873:a0bcab05:a4f0c25d --run --force
# リビルドが走るがどうせしくじるのでスペアディスクになってるやつを削除
mdadm /dev/md0 -f /dev/sdb1

ここまでやってとりあえずRAIDアレイを起動させて出来る限りデータをコピー
